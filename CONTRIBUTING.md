# Contributing to Agency OS

## Workflow
1. Crear issue o tomar una tarea del roadmap.
2. Implementar en rama aislada.
3. Ejecutar pruebas/smoke checks.
4. Actualizar docs si cambia comportamiento.
5. Abrir PR con alcance claro.

## Definition of Done
- código/documentación actualizada
- sin secretos hardcodeados
- output verificable
- siguiente paso documentado

## Commits
Usar mensajes claros tipo:
- feat:
- fix:
- docs:
- chore:
